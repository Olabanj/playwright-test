## Architecture working session (Dmytro)

Iterated on the `playwright-e2e` target architecture and converged on three decisions:

- **Single `playwright.config.ts` with `projects[]`** (api / frontoffice / backoffice) — not multiple config files. Playwright best practice; project-scoped `baseURL` covers front-office (contractor + client) vs back-office (admin only). See `30-decisions/2026-05-15-dmytro-single-config-projects.md`.
- **Domain-agnostic structure** — use `{module-a}`, `{module-b}` placeholders in plan/diagrams; the top-level split (by business module vs by microservice) is deferred until 2-3 real domains exist to test the boundary.
- **Drop 3-lane (`probes/verify`)** — tests live flat under `tests/modules/{module}/{api|ui/frontoffice|ui/backoffice}/`. Selective runs via tags (`--grep @smoke`) and projects (`--project=api`); WIP isolation via git branches. The 3-lane was inherited from the old `test-framework` and is not idiomatic Playwright. `scripts/` moved to repo root for `npx tsx` utilities only. See `30-decisions/2026-05-15-dmytro-drop-3-lane.md`.
